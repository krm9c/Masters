A.3. Installation - target

  1. Install the kernel modules.

    1. On a 2.4.x system copy the modules (aim104*.o) to /lib/modules/kernel-version/arcom/

      And add the lines 

      keep
path[arcom]=/lib/modules/kernel-version/arcom
      to /etc/modules.conf

    2. On a 2.6.x system copy the modules (aim104*.ko) to /lib/modules/kernel-version/extra/

  2. Run (on the target)

    depmod

  3. Create the device node on the target file system by copying mkdevs.sh to the target and running it.

    ./mkdevs.sh

  4. (Optional) If you require the C library:

    1. Copy the lib/libaim104.so.x.y.z shared library to the /usr/lib directory on the target.

  5. (Optional) If you require the Java library:

    1. Copy the java/com/arcom/aim104/libaim104_java.so.x.y.z shared library to the correct directory on the target for the JVM you are using.

      1. For J9 put it in the /opt/wsdd4.0/bin directory (or the directory that contains the j9 executable).

      2. For other JVMs it is usually sufficient to place it in the system library search path (i.e. a directory in /etc/ld.co.conf or in a directory in the LD_LIBRARY_PATH environment variable.

    2. Make the java/aim104.jar JAR file available to the JVM by: copying the JAR to the target and setting classpaths; or including the JAR contents in the application JAR; or, if you are using J9, including the JAR in the application JXE.

  6. (Optional) If you require the GCJ library:

    1. Copy the java/com/arcom/aim104/libaim104_gcj.so.x.y.z shared library to /usr/lib on the target.

  7. (Optional) If you installed any shared libraries:

    1. Run (on the target)

      ldconfig
