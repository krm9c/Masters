A.3. Installation - target

This section describes how to compile and install the AIM104 modules and library onto your target board. However the most recent Arcom development kits include all the precompiled binaries as standard, both on the target and in the Host Environment. We recommend that you use these in preference to building your own. For building kernel modules against a self-built kernel the sources are installed by the host environment in /opt/arcom/src/modules and can be used along with the ael-kernel-build tool as described in the Arcom Embedded Linux technical manual.

Note that more recent development kits are configured with a cross compilation environment known as the Arcom Host Environment, if your development kit is one of these then it will be mentioned in the Technical Manual. In such cases you will need to substitute references to /usr/include and /usr/lib with the correct directories for the host environment, this will be of the form /opt/arcom/TARGET/{include,lib}.

A.1. Compilation

  1. Unpack the source tarball.

    gunzip -c aim104-x.y.z.tar.gz

  2. Change to the top level directory.

    cd aim104-x.y.z

  3. Build the C library and test programs.

    make test-modules
    make lib
    make test-lib

  4. (optional) Build the kernel modules. KERNEL is the path to the kernel tree you wish to build against.

    For a 2.6 kernel build the modules with

    make KSRC=KERNEL modules26

    For a 2.6 kernel build the modules with

    make KSRC=KERNEL modules24

  5. (Optional) If you require the Java or GCJ library:

    make lib-java
    make test-java
    make test-jxe (if you want JXEs for J9)

  6. (Optional) If you require the GCJ library:

    make lib-gcj
    make test-gcj
